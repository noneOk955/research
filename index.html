What is this?

Water that takes care of itself

This project builds a smart water tank that automatically checks if the water quality is good — and fixes it if it's not. No manual testing, no guessing.

Think of it like a thermostat, but for water quality. Just like a thermostat keeps your room at the right temperature, this system keeps your water at the right pH level.

🔬

Continuous Monitoring

A pH sensor checks the acidity of the water every second, displaying it on a small LCD screen.

📡

Ultrasonic Level Detection

A sound-based sensor measures how full the tank is — like a parking sensor, but for water.

⚙️

Automatic Correction

If pH stays out of range for too long, the system drains the old water and refills with fresh water — automatically.

The Science

Understanding pH

pH measures how acidic or basic a liquid is, on a scale from 0 to 14. Most living things need water close to neutral (pH 7) to survive and thrive.

How the system works

💧 Step 1 — Sensing

Every 100 milliseconds (10 times per second!), the ESP32 microcontroller reads two sensors: the pH sensor (PH-4502C) converts water chemistry into a voltage signal, and the HC-SR04 ultrasonic sensor sends a sound pulse and times the echo to calculate how full the tank is — just like a bat echolocating in the dark. The readings are averaged from 20 samples to filter out any electrical noise.

📊 Step 2 — Display

The processed readings are shown on a 16×2 character LCD display. The top row shows the current pH value. If pH is outside the safe limits you've set, a "!" warning symbol appears. The bottom row shows water level as a percentage (0–100%). You can navigate through 6 different screens using the three push buttons.

🔔 Step 3 — Alert

The moment the pH sensor detects a reading outside your configured safe range, an active buzzer starts beeping in a 300ms ON / 700ms OFF pattern. This is non-blocking — the system keeps monitoring and updating the display while the buzzer runs. The buzzer stops automatically as soon as pH returns to the safe range. During calibration mode, the buzzer is silenced to avoid confusion.

⏱️ Step 4 — Confirmation Timer

Before taking any drastic action, the system waits to confirm the problem is real — not just a momentary spike. You can configure this delay from 1 to 30 minutes using Mode 5. If pH recovers to normal at any point during this countdown, the timer resets completely and no action is taken. This prevents the system from draining and refilling your tank due to brief measurement glitches.

🔄 Step 5 — Correction Cycle

After the delay expires with pH still out of range, one correction cycle runs: Pump 2 (Drain) empties the tank down to 10% water level, then Pump 1 (Fill) refills it to 80% with fresh water. After the refill, the system enters a "Done" state and waits for the pH to naturally stabilise before it will trigger again. Only one cycle fires per pH event — it won't keep draining and refilling indefinitely.

Hardware

Components & their functions

Every part has a specific job. Here's what each component does and which pin it connects to.

🧠
Brain

ESP32 Microcontroller

The central processor. Reads sensors, runs all logic, controls the display and pumps, and stores settings even when powered off. Think of it as the brain of the whole system.

ArduinoDroid · 240 MHz · WiFi capable
🔬
Sensor

PH-4502C pH Sensor

Measures the acidity or alkalinity of water by measuring a tiny electrical voltage through the probe. Higher voltage = more acidic. The module amplifies this into a readable signal for the ESP32.

GPIO 34 — ADC input (analog)
📡
Sensor

HC-SR04 Ultrasonic Sensor

Sends a short burst of ultrasound (inaudible to humans) and measures how long the echo takes to return. Closer water surface = shorter echo time = fuller tank. Completely isolated from the water, eliminating electrical interference.

TRIG GPIO 5 · ECHO GPIO 4
🖥️
Output

16×2 I2C LCD Display

Shows 2 rows of 16 characters. Displays real-time pH, water level, mode names, calibration wizards, and status messages. Uses the I2C protocol, so it only needs 2 data wires.

SDA GPIO 23 · SCL GPIO 22 · Address 0x27
⚡
Actuator

4-Channel Relay Module

Acts as an electrically-controlled switch. The ESP32 (3.3V logic) uses it to control high-power pump circuits (12V). The module is active-LOW — sending 0V turns the relay ON. Two channels are used, two are reserved.

Pump1 GPIO 26 · Pump2 GPIO 27
🚰
Actuator

2× Water Pumps

Submersible DC pumps controlled by the relay. Pump 1 (Fill) adds fresh water; Pump 2 (Drain) removes old water. Safety code ensures only one pump can run at a time.

Fill via Relay Ch1 · Drain via Relay Ch2
🔔
Alert

Active Buzzer

Makes a beeping sound when pH goes out of range. "Active" means it generates its own sound frequency internally — just apply voltage and it beeps. Non-blocking code handles the timing pattern.

GPIO 25 — Digital output
🔘
Input

3× Push Buttons

MODE cycles through the 6 screens. SELECT confirms, toggles, or increases a value. NEXT moves to the next item, decreases a value, or saves. All buttons use edge detection — press once, get one action.

MODE 18 · SELECT 19 · NEXT 21
DIY Guide

Build it yourself

Follow these steps in order. No soldering required for basic assembly — just jumper wires and a breadboard.

1

Gather all components

You'll need: ESP32 dev board, PH-4502C module, HC-SR04, 16×2 I2C LCD (0x27), 4-channel relay module, 2 small DC pumps, active buzzer, 3 push buttons, jumper wires, and a breadboard or PCB.

2

Install ArduinoDroid

Download ArduinoDroid from the Play Store. Open it, go to Settings → Board, and select "ESP32 Dev Module". Install the ESP32 board package when prompted.

3

Add the LCD library

Download the "LiquidCrystal I2C" library by Frank de Brabander as a ZIP file. In ArduinoDroid: tap Menu → Sketch → Add Library from ZIP. Select the file.

4

Connect the hardware

Follow the pin table on the right. Connect all components to the ESP32 using jumper wires. Double-check power (3.3V and GND) connections before plugging in.

5

Upload the code

Copy the .ino source code into ArduinoDroid. Connect your phone to the ESP32 via USB-OTG cable. Tap the Upload button. Wait for "Done uploading."

6

Calibrate sensors (Mode 4)

Enter Mode 4 by pressing MODE. Press SELECT to start the wizard. Place the pH probe in a pH 7 buffer solution and follow the on-screen steps. Then calibrate water level (empty then full tank).

7

Set thresholds (Mode 1 & 5)

Go to Mode 1 to set your pH safe range (Lo and Hi values). Go to Mode 5 to set how long pH must stay out of range before the pump correction triggers (1–30 min).

8

Enable automation (Mode 3)

Navigate to Mode 3 and press SELECT to turn Automation ON. The system will now monitor, alert, and self-correct 24/7. Settings are saved to memory automatically.

Real World

Where this system helps

Keeping water at the right pH is critical in many industries and hobbies. Here's where this technology makes a real difference.

Fish keeping
🐠

Aquaculture & Fish Tanks

Fish are extremely sensitive to pH changes. Even a shift of 0.5 pH units can cause stress or death. This system keeps aquariums and fish farms at the ideal 6.5–7.5 range automatically.

Agriculture
🌱

Hydroponics

Plants grown in water need precise pH (5.5–6.5) to absorb nutrients properly. Too acidic or too basic, and plants starve even if nutrients are present. Automated control maximizes crop yield.

Recreation
🏊

Swimming Pools & Spas

Pool water at the wrong pH causes eye irritation, degrades chlorine effectiveness, and damages equipment. Ideal pool pH is 7.2–7.8. This system can alert operators before problems escalate.

Environment
🏭

Water Treatment

Wastewater from factories must be neutralized before release. Regulatory standards require tight pH control. Automated monitoring prevents illegal discharge and protects waterways.

Education
🎓

Science Labs & Research

Experiments requiring consistent conditions benefit from automated pH control. Students can observe chemical processes without constantly monitoring and adjusting manually.

Conservation
🪸

Coral & Marine Systems

Coral reefs are dying partly due to ocean acidification. Reef aquariums require stable pH (8.1–8.4). This system provides the precision needed to keep sensitive marine life healthy.

Common Issues

Troubleshooting guide

Something not working? Here are the most common issues and how to fix them.

pH reads ~11 for vinegar (should be acidic ~2–3)

Critical ▾

This is the inverted slope bug. The original formula used / -0.18 which flips acid and base readings. The corrected code uses / +0.18. Make sure you're using v2.2 of the code. After updating, run the pH calibration wizard (Mode 4) with a known pH 7 buffer solution.

Water level never reaches 100% even when full

Critical ▾

The calibration reference values don't match your tank. Run the level calibration (Mode 4, Steps 3 and 4):

  • Step 3: Empty your tank completely, wait for the ultrasonic reading to stabilise, press SELECT
  • Step 4: Fill your tank to exactly 100%, wait, press NEXT to save

The system will then accurately map the distance range to 0–100%.

LCD shows nothing / stays blank after boot

Medium ▾

Most common cause: wrong I2C address. The code assumes address 0x27. Some displays use 0x3F. Try this I2C scanner sketch to find your display's address, then update the code line: LiquidCrystal_I2C lcd(0x3F, 16, 2);

Also check: SDA is on GPIO 23, SCL is on GPIO 22. The code calls Wire.begin(23, 22);.

Pump turns ON then turns OFF after ~1 second

Critical ▾

This happens when Automation is ON at the same time as Manual mode. The automation state machine keeps calling pumpsOff() every loop, fighting your manual toggle.

  • Either turn Automation OFF (Mode 3, press SELECT) before using Manual mode
  • Or the automation code's IDLE state was incorrectly calling pumpsOff() every cycle — v2.2 fixes this

pH readings fluctuate wildly / interfere with level sensor

Medium ▾

Electrical interference between sensors is common. Solutions in order of effectiveness:

  • Use separate power ground wires for sensors vs relays/pumps ("star grounding")
  • Add 100µF + 100nF capacitors between sensor VCC and GND pins
  • Add flyback diodes (1N4007) across relay coil pins if not present on your module
  • Use shielded cable for the pH probe wire
  • The software already averages 20 samples and uses EMA smoothing to reduce noise

Automation keeps repeating the drain/fill cycle

Medium ▾

The system should run exactly ONE correction cycle per pH event, then wait for pH to recover before triggering again. If it keeps looping, make sure you're using the latest code version with the AUTO_DONE state, which blocks re-triggering until pH actually returns to the normal range.

Settings reset every time power is turned off

Low ▾

EEPROM wasn't initialized properly on first flash. Fix: go to Mode 1, change phLow once (press SELECT), then change it back. This forces an EEPROM write with the magic byte. From then on, all settings will survive power cycles. The magic byte 0xA5 tells the code on boot that valid data exists in EEPROM.

Compile error: "Button not declared" or "readButton declared void"

Low ▾

ArduinoDroid compiles top-to-bottom without forward declarations. The v2.2 code removes all structs and typed enums that caused this issue. If you see this error, make sure you're using the latest .ino file which uses only plain int, float, and bool variable types — no struct or enum : uint8_t syntax.
